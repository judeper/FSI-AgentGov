# Control 1.18: Application-Level Authorization and Role-Based Access Control (RBAC)

## Overview

**Control ID:** 1.18  
**Control Name:** Application-Level Authorization and Role-Based Access Control (RBAC)  
**Regulatory Reference:** FINRA 4511, SOX 302/404, GLBA 501(b), SEC Rule 17a-3/4  
**Setup Time:** 45-60 min  

---

## Governance Levels

### Baseline (Level 1)

Implement role-based access to agents; separate roles for creators, editors, admins, viewers.

### Recommended (Level 2-3)

Least-privilege role assignment with quarterly reviews; automated role removal for departing users.

### Regulated/High-Risk (Level 4)

Fine-grained RBAC with mandatory approval workflows; immutable audit of all role changes.

---

## Verification & Testing

1. Power Platform Admin Center → Environments → Security roles
2. Verify role assignments are least-privilege
3. Test access as different roles (should match expected permissions)
4. Verify audit logs show role changes
EXPECTED: RBAC enforced; users have only required permissions; role changes audited
