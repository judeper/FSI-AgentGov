# Control 1.5: Data Loss Prevention (DLP) and Sensitivity Labels

## Overview

**Control ID:** 1.5  
**Control Name:** Data Loss Prevention (DLP) and Sensitivity Labels  
**Regulatory Reference:** FINRA 4511, SEC Reg S-P, GLBA 501(b), SOX 404  
**Setup Time:** 1-2 hours  

---

## Governance Levels

### Baseline (Level 1)

DLP policies for common sensitive data types + sensitivity labels for key repositories.

### Recommended (Level 2-3)

Strict DLP for shared agents; enforce label-based protection and sharing restrictions.

### Regulated/High-Risk (Level 4)

Strictest DLP + endpoint/session controls; mandatory labels; continuous alerting.

---

## Verification & Testing

1. Microsoft Purview → Data loss prevention → Policies
2. Verify policy is Enabled and includes agent-used locations
3. Create test file with sample PII
4. Try to share (should be blocked)
5. Check audit log for DLP violations
EXPECTED: DLP blocks unauthorized PII sharing; violations logged; labels applied
